# Requirements — Phase 1: Step-level Execution Traceability

## Phase 1 Requirements

| ID | Description | Source |
|----|-------------|--------|
| TRACE-01 | Auto-capture a screenshot after every step completes | D-01 |
| TRACE-02 | Display inline thumbnail (~120px) in step rows; click opens ScreenshotLightbox | D-02 |
| TRACE-03 | Screenshots persist to storage and appear in both live execution and history views | D-03 |
| TRACE-04 | Expandable step rows showing tool call details, LLM reasoning, timing breakdown, error stack traces | D-04 |
| TRACE-05 | Expanded detail organized as tabbed sections: Tools, Reasoning, Timing, Errors | D-05 |
| TRACE-06 | Logs appear inline within LiveStepTimeline expanded step detail | D-06 |
| TRACE-07 | Tool call events stream in real-time via SSE while step is still running | D-07 |
| TRACE-08 | Optional Playwright video recording toggle before run start (default off) | D-08 |
| TRACE-09 | Video playback appears in RunDetailView as a player section | D-09 |
