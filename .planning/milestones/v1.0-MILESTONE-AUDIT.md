## Cross-Phase Integration Gaps

1. Parser assertions are not enforced in the real live-MCP execution path.
2. Screenshot capture does not flow from tool instrumentation into persisted evidence and vision validation.
3. Browserbase remains selectable in the UI despite lacking an executable runtime configuration.
4. `retryWithNewSnapshot()` exists but is still disconnected from the production stale-ref path.
5. The CLI bypasses `runManager`, so CLI benchmark behavior diverges from the actual product path.

## Broken Or Risky Flows

1. A live UI run with a failing `Then` step can still report success without applying the independent assertion result.
2. A live UI run expecting hallucination flags and screenshots can degrade to `NEEDS_REVIEW` solely because no screenshot evidence exists.
3. The default multi-MCP UI run is risky because Browserbase is selected by default but not executable.
4. The CLI headless benchmark flow is not a real end-to-end benchmark path yet.
5. Run-detail CSV export is still inconsistent with the per-MCP scorecard promise.
