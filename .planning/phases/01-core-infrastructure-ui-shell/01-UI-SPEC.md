## Aesthetic Directive

**Reference:** Claude Code web app (claude.ai/code), PostHog, Datadog, Linear — developer tooling tier.

**Audience:** Developers and QA engineers running MCP benchmark runs. The UI must communicate data density and technical precision, not consumer-app warmth.

**Anti-patterns banned in this phase (vibecoded markers):**
- `backdrop-filter: blur()` — remove from `.panel` and all surfaces
- Decorative `radial-gradient` / `linear-gradient` on `body` background — remove entirely
- `border-radius` above `6px` on panels and cards (the current `1rem` / `rounded-2xl` is banned)
- `box-shadow: 0 14px 40px ...` — remove; no decorative shadows anywhere
- `rounded-full` on non-pill interactive elements (SSE status badge stays; panel corners do not)
- Gradient text
- Animation beyond functional feedback (loading spinner OK; entrance animations not OK)

**Allowed purposeful styling:**
- 1px solid borders at low opacity (surfaces feel structured, not floating)
- Tight `border-radius: 4px` or `6px` on containers; `2px` on inline chips
- Monospace font for all code, feature text, token counts, IDs
- Semantic color tokens only on status indicators, never decoratively
- Focus rings via `box-shadow: 0 0 0 2px` on interactive elements — functional only

## Layout Contract

**Shell structure (from CONTEXT.md D-02 + REQUIREMENTS.md UI-09):**

```
+--250px sidebar--+-------- main content area ----------+
| Logo / app name |                                      |
| [New Run]       |  <view: scenario editor or live run> |
| [Run History]   |                                      |
| [Settings]      |                                      |
|                 |                                      |
| theme toggle    |                                      |
+-----------------+--------------------------------------+
```

- Sidebar width: 240px fixed, not collapsible in Phase 1
- Sidebar background: `--app-sidebar` (see Color section below) — distinct from main content background
- Sidebar nav items: 36px tall, 8px horizontal padding, `border-radius: 4px` on hover/active state
- Active nav item: accent-tinted background + left 2px accent border
- Main content area: `max-width: 1400px`, padding `24px` all sides
- Minimum supported viewport: 1280px wide
- No mobile breakpoints in Phase 1 (UI-09: desktop 1280px+ primary target)

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline tight spacing |
| sm | 8px | Compact element spacing, nav item padding |
| md | 16px | Default element spacing, panel inner padding |
| lg | 24px | Section padding, major inner gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in Phase 1) |

**Exceptions:**
- MCP selector row height: 40px (tap-safe, readable at density)
- Sidebar nav item height: 36px
- Modal padding: 20px (lg minus 4px — keeps modal compact without feeling cramped)

## Typography

| Role | Size | Weight | Line Height | Font Family | Notes |
|------|------|--------|-------------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Geist Variable | Default prose, log entries |
| Label | 12px | 600 (semibold) | 1.4 | Geist Variable | Field labels, section titles, nav items |
| Code / Mono | 13px | 400 (regular) | 1.6 | `ui-monospace, "Cascadia Code", "Fira Code", monospace` | Gherkin textarea, token counts, run IDs, log timestamps |
| Heading | 16px | 600 (semibold) | 1.2 | Geist Variable | Panel headings, modal titles |

**Rules:**
- Exactly 2 weights used: 400 and 600. No 500, no 700.
- No font size below 11px in the rendered UI (11px is reserved for status chip labels only).
- Letter-spacing: `-0.01em` on body and headings (Geist Variable renders tighter without it).
- Uppercase labels use `letter-spacing: 0.10em` and `font-size: 11px` at weight 600.

## Interaction States

Every interactive element must have exactly these states declared:

| Element | Default | Hover | Focus | Active / Selected | Disabled |
|---------|---------|-------|-------|-------------------|---------|
| Primary button | accent bg | accent-strong bg | accent focus ring | pressed (opacity 0.9) | opacity 0.5, no-cursor |
| Secondary button | transparent | panel-strong bg | border-strong | — | opacity 0.5 |
| Field input | border | border-strong | accent focus ring | — | opacity 0.6 |
| MCP selector row | panel bg | panel-strong + border-strong | accent focus ring | accent left border + tinted bg | — |
| Sidebar nav item | transparent | panel bg | focus ring | accent left border + tinted bg | — |
| Checkbox | unchecked border | — | focus ring | accent fill | — |

Focus rings: `box-shadow: 0 0 0 2px color-mix(in srgb, var(--app-accent) 30%, transparent)`. Never `outline: none` without a visible replacement.
