    The Playwright MCP server respects the `PLAYWRIGHT_RECORD_VIDEO` or similar env var, OR the video recording is controlled via Playwright browser context options. Since the MCP server manages the browser context, we need to pass this as an environment variable when spawning the MCP process.

    In `executeMcpRun`, where `McpProcessManager` is instantiated or `preflight` is called, pass the recordVideo flag:
    ```typescript

    }
    ```

    If `McpProcessManager` doesn't support extra env vars yet, add the video dir to the process environment directly. The exact mechanism depends on McpProcessManager's API — read its spawn method to determine how to pass extra env.

    5. After the run completes, if video was recorded, emit a `video_available` SSE event:
    ```typescript
    if (session.config.recordVideo) {
      // Check for video files in the video directory
      const videoDir = `${SCREENSHOT_DIR}/videos/${session.id}`;
      try {
        const fs = await import("node:fs/promises");
        const files = await fs.readdir(videoDir);
        const videoFile = files.find(f => f.endsWith(".webm") || f.endsWith(".mp4"));
        if (videoFile) {
          this.emit(session, "video_available", {
            runId: session.id,
            videoPath: `${videoDir}/${videoFile}`,
            videoUrl: `/api/videos/${encodeURIComponent(session.id)}/${encodeURIComponent(videoFile)}`,
          });
        }
      } catch {
        // No video directory or files
      }
    }
    ```

    6. Add a GET route for serving video files. In `src/server/index.ts`, add:
    ```typescript
    server.get("/api/videos/:runId/:filename", async (request, reply) => {
      const { runId, filename } = request.params as { runId: string; filename: string };
      const videoPath = path.join(SCREENSHOT_DIR, "videos", runId, filename);
      // Validate path doesn't escape
      const resolved = path.resolve(videoPath);
      if (!resolved.startsWith(path.resolve(SCREENSHOT_DIR))) {
        return reply.code(403).send({ error: "Forbidden" });
      }
      return reply.sendFile(filename, path.join(SCREENSHOT_DIR, "videos", runId));
    });
    ```

    NOTE: This requires fastify-static or manual stream. Use `reply.type("video/webm").send(createReadStream(videoPath))` as fallback if fastify-static isn't installed.
