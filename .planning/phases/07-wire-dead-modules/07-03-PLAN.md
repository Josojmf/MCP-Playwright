# Plan 3: Wire assertionsRunner.runAssertion() for Then steps

## Objective

Call `runAssertion()` from `src/server/validation/assertionsRunner.ts` after each `Then` step in the orchestrator step loop, so that Playwright assertions run independently of what the MCP reported, and the result propagates to step status.

## Context

**Dead module:** `src/server/validation/assertionsRunner.ts` exports `runAssertion(assertion, context)` which:
- Takes a `TranslatedAssertion` (from `ParsedStep.assertion`) and an `AssertionContext`
- Executes the Playwright assertion independently
- Returns `AssertionResult { status: 'passed' | 'failed', message?, stack? }`

**Parser already populates assertions:** `src/server/parser/index.ts` line 119 sets `assertion: currentCanonical === 'then' ? translateAssertion(step.text) : undefined` on each `ParsedStep`. So `step.assertion` is already available on Then steps.

**Current call site:** `OrchestratorService.runScenario()` processes steps at lines 78-174 but never checks `step.assertion`. After a successful LLM call (line 118-131), it always yields `"passed"`.

**What needs to change:** After the LLM call succeeds for a `Then` step, check if `step.assertion` exists. If so, call `runAssertion(step.assertion, {})`. If the assertion fails, override the step status to `"failed"` regardless of what the LLM reported.

## Implementation Steps

1. **Add imports** at top of `src/server/orchestrator/OrchestratorService.ts`:
   ```typescript
   import { runAssertion, AssertionResult } from "../validation/assertionsRunner";
   ```

2. **Add assertion check after successful LLM response** (after line 113, before the yield at line 119). Insert:
   ```typescript
   // VALID-02: Run independent assertion for Then steps
   let assertionOverride: AssertionResult | null = null;
   if (step.canonicalType === "then" && step.assertion) {
     assertionOverride = await runAssertion(step.assertion, {});
   }

   const stepStatus = assertionOverride?.status === "failed" ? "failed" : "passed";
   const stepMessage = assertionOverride?.status === "failed"
     ? `Assertion independiente falló: ${assertionOverride.message}`
     : `Paso completado exitosamente (${response.usage.totalTokens} tokens)`;
   ```

3. **Update the yield** at line 119 to use `stepStatus` and `stepMessage`:
   ```typescript
   yield this.createStepResult(
     ctx.mcpConfig.id,
     step,
     scenario,
     stepIndex,
     stepStatus,  // was: "passed"
     {
       input: response.usage.promptTokens,
       output: response.usage.completionTokens,
       total: response.usage.totalTokens,
     },
     latencyMs,
     stepMessage,  // was: `Paso completado exitosamente (${response.usage.totalTokens} tokens)`

   ```

4. **Update `ParsedStep` import** if not already imported (check the import from `../parser`). The `assertion` field is of type `TranslatedAssertion | undefined`, already defined in `ParsedStep`. Make sure the OrchestratorService has access to the `ParsedStep` type (it uses `step.keyword`, `step.text`, `step.canonicalType` already -- `step.assertion` should be accessible).

5. **Add test** in `OrchestratorService.test.ts` for assertion override:
   ```typescript
   test("Then step with assertion overrides status to failed when assertion fails", async () => {
     const mockProvider = new MockLLMProvider();
     const orchestrator = new OrchestratorService(mockProvider);
     const budget = new TokenBudget({ hardCapTokens: 10000, warnThresholdRatio: 0.8 }, () => {});

     const thenStep: ParsedStep = {
       keyword: "Then",
       canonicalType: "then",
       text: "the page URL should be \"https://wrong-url.com\"",
       assertion: {
         original: "the page URL should be \"https://wrong-url.com\"",
         playwrightCall: "expect(page).toHaveURL('https://wrong-url.com')",
         pattern: "url",
         confidence: 1.0,
       },
     };

     const scenario: ScenarioPlan = {
       id: "scenario-assertion",
       name: "Assertion Test",
       tags: [],
       steps: [thenStep],
     };

     const ctx: RunContext = {
       runId: randomUUID(),
       scenario,
       mcpConfig: { id: "test-mcp", provider: { provider: "openai", model: "gpt-4" } as ProviderConfig },
       conversationHistory: [],
       tokenBudget: budget,
       abortSignal: new AbortController().signal,
     };

     const results: StepResult[] = [];
     for await (const result of orchestrator.runScenario(scenario, ctx)) {
       results.push(result);
     }

     assert.equal(results.length, 1);
     // The assertion runner with mock expect will fail for wrong URL
     // The step should be marked failed due to assertion, not passed from LLM
     assert.ok(results[0].status === "failed" || results[0].message.includes("Assertion"));
   });
   ```
