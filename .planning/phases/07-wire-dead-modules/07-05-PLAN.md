# Plan 5: Wire TokenBudget.checkBudget() before each LLM request

## Objective

Guard every `provider.complete()` call in `OrchestratorService` with a `ctx.tokenBudget.checkBudget()` call, so runs that would exceed the token cap are blocked before the LLM request is made.

## Context

**Dead module method:** `TokenBudget.checkBudget(estimatedNext?)` in `src/shared/harness/TokenBudget.ts` (line 42). It throws `BudgetExceededError` if `usedTokens + estimatedNext > hardCap`. This method exists and is tested but is never called from the orchestrator -- only `addUsage()` is called (after the LLM response, line 116).

**Current call site:** `OrchestratorService.runScenario()` line 96-101 calls `provider.complete(request)` inside `withTimeout()`. There is no budget pre-check before this call.

**What needs to change:** Add `ctx.tokenBudget.checkBudget()` immediately before the `withTimeout(provider.complete(...))` call, inside the step loop. If the budget is exceeded, the `BudgetExceededError` propagates up through the existing catch block, which will mark the step as failed and (since `isFatalStepError` won't match) continue or stop based on `continueOnError`. We should treat budget exceeded as a fatal error to abort the run.

## Implementation Steps

1. **Add BudgetExceededError import** at top of `src/server/orchestrator/OrchestratorService.ts`:
   ```typescript
   import { BudgetExceededError } from "../../shared/harness/TokenBudget";
   ```

2. **Add budget check before LLM call** (before line 96):
   ```typescript
   // Guard: check token budget before making LLM request (INFRA-05)
   ctx.tokenBudget.checkBudget();
   ```
   Insert this line right before the `const response = await withTimeout(...)` call (current line 96).

3. **Add BudgetExceededError to fatal error detection.** Update `isFatalStepError` (line 245) to also catch budget errors, OR add a separate check in the catch block. The cleaner approach is to add a type check in the catch block at line 134:
   ```typescript
   } catch (error) {
     const latencyMs = Date.now() - stepStartTime;

     // Budget exceeded is fatal -- abort the entire run
     if (error instanceof BudgetExceededError) {
       yield this.createStepResult(
         ctx.mcpConfig.id, step, scenario, stepIndex,
         "aborted",
         { input: 0, output: 0, total: 0 },
         latencyMs,
         `Presupuesto de tokens excedido: ${error.message}`,
         []
       );
       return; // Exit the generator entirely
     }

     // ... rest of existing error handling ...
   ```

4. **Add test** in `OrchestratorService.test.ts` for budget pre-check:
   ```typescript
   test("runScenario aborts when token budget is exceeded before LLM call", async () => {
     const mockProvider = new MockLLMProvider();
     const orchestrator = new OrchestratorService(mockProvider);
     const tightBudget = new TokenBudget(
       { hardCapTokens: 100, warnThresholdRatio: 0.8 },
       () => {}
     );
     // Pre-fill budget to just under cap
     tightBudget.addUsage(101);

     const ctx: RunContext = {
       runId: randomUUID(),
       scenario: oneStepScenario,
       mcpConfig: testMCPConfig,
       conversationHistory: [],
       tokenBudget: tightBudget,
       abortSignal: new AbortController().signal,
     };

     const results: StepResult[] = [];
     for await (const result of orchestrator.runScenario(oneStepScenario, ctx)) {
       results.push(result);
     }
     assert.equal(results.length, 1);
     assert.equal(results[0].status, "aborted");
     assert.ok(results[0].message.includes("Presupuesto de tokens excedido") || results[0].message.includes("budget"));
   });
   ```
