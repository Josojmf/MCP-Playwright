# Phase 08 — UI Design Contract

> Visual and interaction contract for Phase 08: Real MCP Process Protocol & Integration Fixes.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Scope Declaration

**Phase 08 introduces zero new UI surfaces.** CONTEXT.md explicitly states: "No new user-visible features; no UI changes; no new MCP servers added."

The four changes in this phase are entirely server-side and CLI:
1. `McpProcessManager` — real stdio transport replacing stub process
2. CLI `--provider` flag — real `createProvider()` factory replacing mock
3. `LoopDetector` fingerprinting — call-site fix in `runManager.ts`
4. `estimateRun()` pricing — `resolvePricing()` replacing hardcoded constants

This UI-SPEC documents the **existing design contract** that all prior and future phases must conform to. No new components, copywriting, or interactions are introduced by this phase. Executors must not create any new UI files in Phase 08.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element separation |
| sm | 8px | Compact element spacing, chip padding |
| md | 16px | Default element spacing, panel padding |
| lg | 24px | Section padding, card internal spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page-level margins |
| 3xl | 64px | Page-level top spacing |

Exceptions:
- Sidebar nav item height: 36px (established in `.sidebar-nav-item`)
- Field input height: 36px (established in `.field-input`)
- MCP option min-height: 40px (established in `.mcp-option`)
- Touch target minimum: 44px (accessibility, applies to interactive controls)

Source: src/client/index.css — all values verified against existing `.panel`, `.field`, `.sidebar-nav-item`, `.metric-card` utility classes.

## Typography

All text uses `Geist Variable`, `sans-serif` at `letter-spacing: -0.01em`.

| Role | Size | Weight | Line Height | Token / Class |
|------|------|--------|-------------|---------------|
| Body | 14px (0.875rem) | 400 | 1.5 | `.field-input`, `.app-soft-button` |
| Label | 12px (0.75rem) | 600 | 1.2 | `.section-title`, `.field-label`, `.chip` |
| Heading | 18px (1.125rem) | 600 | 1.2 | `EmptyState` h3 at `text-lg font-semibold` |
| Metric / micro | 11px | 600 | 1.2 | `.metric-label`, `.status-chip` (uppercase, tracked) |

Two weights in use: **400 (regular)** and **600 (semibold)**. No other weights are permitted.

Uppercase label pattern: `text-transform: uppercase; letter-spacing: 0.1em–0.14em` — applied only to `.section-title`, `.field-label`, `.metric-label`, `.status-chip`, `.app-badge`.

## Color

All values are CSS custom properties defined in `src/client/index.css`. Light and dark modes are fully declared.

| Role | Light Value | Dark Value | Usage |
|------|-------------|------------|-------|
| Dominant (60%) | `--app-bg` #f3f6fb | `--app-bg` #050912 | Page background, app shell |
| Secondary (30%) | `--app-panel` #ffffff / `--app-sidebar` #e9eef6 | `--app-panel` #0f1725 / `--app-sidebar` #0c121c | Cards, panels, sidebar, nav |
| Accent (10%) | `--app-accent` #0d9488 | `--app-accent` #2dd4bf | Reserved elements only (see below) |
| Destructive | `--app-danger` #dc2626 | `--app-danger` #f87171 | Destructive actions, error status chips only |

Accent (`--app-accent`) is **reserved exclusively for**:
- Primary action buttons (`.app-primary-button` gradient)
- Active sidebar nav item left border + background tint
- Selected MCP option left border + background tint
- Focus ring on interactive controls (2px ring at 30–34% opacity)
- Running status chip background/border
- `chip` component background/border

Semantic color palette (status indicators only — never for decoration):
- Success: `--app-success` #059669 / dark #34d399
- Warning: `--app-warning` #d97706 / dark #fbbf24
- Danger: `--app-danger` #dc2626 / dark #f87171
- Info: `--app-info` #2563eb / dark #60a5fa

## Copywriting Contract

Phase 08 introduces no new copy. The following contract documents existing copy patterns that must be maintained by all phases.

| Element | Copy | Source |
|---------|------|--------|
| Primary CTA | "Start Run" | Established pattern (run initiation) |
| Empty state heading | "No runs yet" | RunHistoryList empty state |
| Empty state body | "Run your first benchmark to see results here." | RunHistoryList empty state |
| Error state — unknown model | `Unknown pricing for provider "{provider}" model "{model}". Check PRICING_TABLE in src/shared/pricing/table.ts.` | D-18 in CONTEXT.md, Claude's discretion |
| Error state — missing --provider | `Error: --provider is required. Supported: openrouter \| openai \| azure \| anthropic` followed by env var table | D-12 in CONTEXT.md, confirmed in RESEARCH.md Pattern 5 |
| Loop abort status label | "aborted" | D-16 in CONTEXT.md — distinct from "failed" |
| Destructive confirmation | Not applicable — Phase 08 introduces no destructive UI actions | — |

**CLI error message for missing --provider (exact copy, from RESEARCH.md Pattern 5):**
```
Error: --provider is required. Supported: openrouter | openai | azure | anthropic
Required env vars:
  openrouter → OPENROUTER_API_KEY
  openai     → OPENAI_API_KEY
  azure      → AZURE_OPENAI_API_KEY + AZURE_OPENAI_ENDPOINT
  anthropic  → ANTHROPIC_API_KEY
```

**Error message for unknown pricing model (exact copy, from D-18 + RESEARCH.md Pattern 7):**
```
Unknown pricing for provider "{provider}" model "{model}". Check PRICING_TABLE in src/shared/pricing/table.ts.
```
