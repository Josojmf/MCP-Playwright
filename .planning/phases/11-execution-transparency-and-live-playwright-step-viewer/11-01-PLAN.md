# Plan 11-01: Live MCP Column Grid with Screenshot Lightbox and Suspicious Step Flagging

## Objective

Replace the current stacked `progressByMcp` card grid in App.tsx (lines 898-995) with a side-by-side column layout. Each MCP gets its own column showing a header with status chip, progress bar, the latest screenshot (always visible, clickable to open a lightbox modal), and a scrollable step list with auto-scroll. Suspicious steps (hallucinated / needsReview) get visual flagging with colored backgrounds and labels per D-12. On narrow screens or 3+ MCPs, the layout degrades to shadcn Tabs.

Purpose: Delivers the "live camera feed" experience per D-05 and closes the folded todo "Mostrar screenshot o video por paso". This is the foundation that Plan 11-02 (scorecard) builds on top of.

Output: Four new component files, two shadcn components installed, App.tsx migrated.

## Steps

Run:
```bash

npx shadcn add tabs
```

Verify both components exist at `src/client/components/ui/dialog.tsx` and `src/client/components/ui/tabs.tsx`.

- Image: `max-width: 100%; max-height: calc(90vh - 80px); object-fit: contain`.
- Close button: shadcn DialogClose at top-right, ghost variant.

### Step 3: Create McpColumn component

Create `src/client/components/run/McpColumn.tsx`.

Props:

```typescript
interface McpColumnProps {
  mcpId: string;
  progress: ProgressState;                    // from App.tsx state
  steps: StepEvidence[];                      // from stepEvidenceByMcp[mcpId]
  lastScreenshotId: string | null;            // from lastScreenshotByMcp[mcpId]
  onScreenshotClick: (title: string, url: string) => void;  // opens lightbox
}
```

Import `ProgressState` and `StepEvidence` types from App.tsx (they will need to be exported or moved to a shared types file -- export them from App.tsx for now, refactor later if needed).

Implementation per 11-UI-SPEC.md "MCP Column (live view)" spec:

- **Container:** `border: 1px solid var(--app-border); background: var(--app-panel); border-radius: 6px; padding: 16px; display: flex; flex-direction: column; gap: 8px; min-width: 320px`.
- **Header row:** flex between MCP name (14px weight 600 `--app-fg-strong`) and status chip (existing `status-chip` + `statusChipClass` pattern from App.tsx).
- **Progress bar:** 4px height (thinner than LiveStepTimeline's 8px), `--app-track` bg, `--app-accent` fill, 300ms transition.
- **Progress counter:** `{completedSteps} de {totalSteps} pasos` at 11px `--app-muted`.
- **Screenshot area:** Fixed 168px height. If `lastScreenshotId` is truthy, render `<img>` with `src=/api/screenshots/${lastScreenshotId}`, `object-fit: cover`, `width: 100%`, `height: 100%`, `cursor: pointer`. On click call `onScreenshotClick` with title `"{mcpId} . Paso {steps.length}"` and URL. If no screenshot, render placeholder div: same 168px height, `background: var(--app-panel-strong)`, centered text `Sin screenshot todavia.` at 11px `--app-muted`. Both have `border: 1px solid var(--app-border); border-radius: 4px`.
- **Step list:** `overflow-y: auto; max-height: 480px; display: flex; flex-direction: column; gap: 8px`. Use a `useRef` + `useEffect` to auto-scroll to bottom (`scrollTop = scrollHeight`) whenever `steps.length` changes. Render steps in order (oldest first -- note `stepEvidenceByMcp` prepends new items, so reverse the array before rendering).
- **Step row:** For each step, render a row with:
  - Status icon: `CheckCircle2` for passed (green), `XCircle` for failed (red) from lucide-react.
  - Step text at 14px `--app-fg`.
  - Tokens + latency at 13px monospace `--app-muted`.
  - **Suspicious step flagging (D-12):** The SSE events `step_passed` and `step_failed` already emit `hallucinated` and `needsReview` booleans (see App.tsx lines 467-468). However, `StepEvidence` does not currently store these fields. **Extend the `StepEvidence` interface** in App.tsx to add `hallucinated?: boolean` and `needsReview?: boolean`. Update the `appendStepEvidence` calls in `step_passed` (line 496-505) and `step_failed` (line 548-557) handlers to pass these values through.
  - If `step.hallucinated === true`: row gets `background: color-mix(in srgb, var(--app-danger) 10%, var(--app-panel)); border-left: 3px solid color-mix(in srgb, var(--app-danger) 30%, transparent)`. Show `XCircle` icon in red + label `[HALLUCINATED]` at 11px weight 600 uppercase `--app-danger`.
  - If `step.needsReview === true` (and not hallucinated): row gets `background: color-mix(in srgb, var(--app-warning) 10%, var(--app-panel)); border-left: 3px solid color-mix(in srgb, var(--app-warning) 30%, transparent)`. Show `AlertTriangle` icon in yellow + label `[NEEDS REVIEW]` at 11px weight 600 uppercase `--app-warning`.
  - Normal steps: `background: var(--app-panel); border: 1px solid var(--app-border); border-radius: 4px; padding: 8px 16px`.

### Step 4: Create McpColumnGrid component

Create `src/client/components/run/McpColumnGrid.tsx`.

Props:

```typescript
interface McpColumnGridProps {
  progressByMcp: Record<string, ProgressState>;
  stepEvidenceByMcp: Record<string, StepEvidence[]>;
  lastScreenshotByMcp: Record<string, string | null>;
  isRunning: boolean;
  onAbort: () => void;
}
```

Implementation:
- Maintain local state for lightbox: `{ open: boolean; title: string; url: string }`.
- Count MCPs: `const mcpIds = Object.keys(progressByMcp)`.
- **Grid mode (default):** When `mcpIds.length <= 2` AND container width >= 640px, render `display: grid; grid-template-columns: 1fr 1fr; gap: 16px` (per D-01, D-02). For 1 MCP, use `grid-template-columns: 1fr`. Render one `McpColumn` per MCP.
- **Tab fallback (D-03):** When `mcpIds.length >= 3` OR container width < 640px, render shadcn `Tabs` with one `TabsTrigger` per MCP and one `TabsContent` per MCP, each containing an `McpColumn`. Tab styling per 11-UI-SPEC.md: `TabsList` bg `--app-panel-strong`, border `--app-border`, border-radius 4px, padding 4px. `TabsTrigger` 36px height, 14px font, `--app-muted` text, active: `--app-panel` bg, `--app-fg-strong` text, weight 600.
- Use a simple width check: use `useRef` on the container + `ResizeObserver` to track width. Default to grid if no measurement yet.
- **Abort button:** Below the grid/tabs, centered, full width. Same destructive styling as LiveStepTimeline abort button. Only show when `isRunning === true`. Label: `Abortar Ejecucion`.
- **ScreenshotLightbox:** Render at bottom of component, driven by local lightbox state.
