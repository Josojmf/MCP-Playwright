# Plan 11-02: Post-Run Scorecard with Step Replay and Suspicious Step Flagging

## Objective

Create a post-run scorecard component that replaces the live column view when a run reaches `completed` or `aborted` state. The scorecard shows a metrics summary table (one row per MCP: pass rate, hallucination count, tokens+cost, avg latency) followed by a step replay accordion where each MCP's steps are expandable to reveal screenshots inline. Suspicious step flagging carries over from the live view with the same visual treatment (D-13).

Purpose: Closes the scorecard deliverable (D-08 through D-11) and provides the at-a-glance comparison table that is the primary output artifact of a benchmark run.

Output: `RunScorecard.tsx` component, App.tsx wired with conditional render (running -> columns, completed/aborted -> scorecard).

### Step 3: Extract shared step row flagging logic

Both `McpColumn` (Plan 11-01) and `RunScorecard` (this plan) need the same suspicious step visual treatment. To avoid duplication:

Create `src/client/components/run/StepFlagStyles.ts` (a pure utility, no JSX):

```typescript
export function getStepFlagClasses(step: { hallucinated?: boolean; needsReview?: boolean }): {
  containerClass: string;
  icon: "hallucinated" | "needsReview" | null;
  label: string | null;
} {
  if (step.hallucinated) {
    return {
      containerClass: "bg-[color-mix(in_srgb,var(--app-danger)_10%,var(--app-panel))] border-l-[3px] border-l-[color-mix(in_srgb,var(--app-danger)_30%,transparent)]",
      icon: "hallucinated",
      label: "[HALLUCINATED]",
    };
  }
  if (step.needsReview) {
    return {
      containerClass: "bg-[color-mix(in_srgb,var(--app-warning)_10%,var(--app-panel))] border-l-[3px] border-l-[color-mix(in_srgb,var(--app-warning)_30%,transparent)]",
      icon: "needsReview",
      label: "[NEEDS REVIEW]",
    };
  }
  return { containerClass: "", icon: null, label: null };
}
```

Then update both `McpColumn.tsx` and `RunScorecard.tsx` to import and use `getStepFlagClasses` instead of inline flag logic. This ensures D-12 and D-13 are visually identical.
