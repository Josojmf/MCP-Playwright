## Aesthetic Directive

**Inherited from Phase 1 — unchanged.** All anti-patterns and allowed styling rules from 01-UI-SPEC.md apply in full.

Key rules that affect Phase 11 components specifically:

- The live column layout and scorecard panels use `border: 1px solid var(--app-border)`, `background: var(--app-panel)`, `border-radius: 6px`. No backdrop-filter. No decorative box-shadow.
- The screenshot lightbox modal overlay is `background: rgba(0,0,0,0.55)` — flat, no blur.
- Step status colors use semantic tokens only (`--app-success`, `--app-danger`, `--app-warning`). Never used decoratively.
- Transition from live view to scorecard: **instant swap** — no fade, no slide animation. RunState change drives a conditional render with no CSS transition on the container. Functional animations (spinner, progress bar fill) are permitted as before.
- Tab component (MCP fallback): uses `border-radius: 4px` on tab triggers; no pill/rounded-full shape.

## Layout Contract

### Live Run Section — Side-by-Side Column View

Replaces the current `progressByMcp` stacked card grid in `App.tsx`.

```
+--- main content area -------------------------------------------+
|  [MCP Column: playwright]          [MCP Column: puppeteer]      |
|  +------------------------------+  +---------------------------+|
|  | MCP header + status chip     |  | MCP header + status chip  ||
|  +------------------------------+  +---------------------------+|
|  | Progress bar                 |  | Progress bar              ||
|  +------------------------------+  +---------------------------+|
|  | Screenshot (latest, always   |  | Screenshot (latest)       ||
|  | visible — 168px tall)        |  | 168px tall                ||
|  +------------------------------+  +---------------------------+|
|  | Step list (scrollable)       |  | Step list (scrollable)    ||
|  | auto-scrolls to latest step  |  | auto-scrolls to latest    ||
|  +------------------------------+  +---------------------------+|
|  [Abortar Ejecución] (centered, full-width across columns)      |
+-----------------------------------------------------------------+
```

- **Column grid:** `display: grid; grid-template-columns: 1fr 1fr; gap: 16px` for 2 MCPs.
- **For 3+ MCPs or viewport < 1280px:** render `Tabs` component (shadcn official) instead of grid. One tab per MCP. Same column content per tab.
- **Column min-width:** 320px per column. Below 640px total width per column, switch to tabs.
- **Column scroll:** each column's step list section has `overflow-y: auto; max-height: 480px`. Auto-scrolls to bottom on each new step event via `scrollTop = scrollHeight`.
- **Screenshot area:** `height: 168px; width: 100%; object-fit: cover; border: 1px solid var(--app-border); border-radius: 4px`. Clickable to open lightbox modal.
- **MCP column header:** MCP name at 14px weight 600 `--app-fg-strong` + status chip at right.
- **Screenshot empty state (no screenshot yet):** placeholder div at same 168px height, `background: var(--app-panel-strong); border: 1px solid var(--app-border); border-radius: 4px`. Contains centered text `Sin screenshot todavía.` at 11px `--app-muted`.

### Scorecard Section (post-run, inline transition)

Replaces the live column view when `runState === "completed"` or `runState === "aborted"`. Same panel, different content — no route change.

```
+--- Scorecard panel -------------------------------------------+
|  Section title: RESULTADOS DEL RUN                            |
|  +-- Metrics table (full width) ---------------------------+  |
|  | MCP | Pass Rate | Hallucinations | Tokens+Cost | Latency |  |
|  | ... |    ...    |      ...       |    ...      |   ...   |  |
|  +------------------------------------------------------+  |
|  Section title: REPLAY DE PASOS                              |
|  +-- MCP accordion / expandable list -------------------+    |
|  | [MCP name] ▼                                         |    |
|  |   Step row 1 (expandable → screenshot inline)        |    |
|  |   Step row 2 ...                                     |    |
|  +------------------------------------------------------+    |
+--------------------------------------------------------------+
```

- **Metrics table:** standard HTML `<table>` with existing `metric-card` / `metric-label` / `metric-value` class patterns extended to table cells. No third-party table component.
- **Table header row:** 11px uppercase weight 600 `--app-muted` cells.
- **Table body rows:** one row per MCP. 14px weight 400 `--app-fg`. Alternating row background: even rows `--app-panel`, odd rows `--app-panel-strong`.
- **Metrics table columns:** MCP name | Pass Rate (%) | Hallucinations | Tokens + USD cost | Avg latency (ms).
- **Step replay defaults to collapsed per MCP.** User clicks MCP name row to expand/collapse the step list for that MCP. Chevron icon (lucide `ChevronDown` / `ChevronRight`) at 16px indicates state.
- **Step row in replay (collapsed, one line):** step index + step text + status badge + tokens + latency. Clicking expands to show screenshot inline (same `screenshotsByStep` pattern as RunDetailView).
- **Expanded step screenshot:** `width: 100%; max-height: 240px; object-fit: contain; border: 1px solid var(--app-border); border-radius: 4px; margin-top: 8px`. Link to full-res opens in new tab (`target="_blank"`).

## Spacing Scale

Inherited from Phase 1 — no changes.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, inline tight spacing |
| sm | 8px | Compact element spacing, nav item padding |
| md | 16px | Default element spacing, panel inner padding |
| lg | 24px | Section padding, major inner gaps |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions for Phase 11:**

| Element | Value | Reason |
|---------|-------|--------|
| Column gap | 16px (md) | Tighter than xl — columns are already wide |
| Screenshot thumbnail height | 168px | Fixed aspect ratio for 16:9 browser screenshots |
| Step row padding (replay) | 10px 12px | Denser than md — scorecard rows need to be scannable, not spacious |
| Lightbox modal padding | 20px | Inherited from Phase 1 modal spec |
| Tab trigger height | 36px | Matches sidebar nav item height for visual consistency |

## Typography

Inherited from Phase 1 — no new sizes or weights added.

| Role | Size | Weight | Line Height | Font Family | Notes |
